# Android SAST

Android SAST is an experimental Android app for inspecting APK files for security issues. It decompiles APKs to smali, runs deterministic rules, optionally adds AI-based discovery and enrichment, then exports findings as reports.

## Features

- APK selection and analysis from the Android app
- Smali-based rule scanning
- Optional AI enrichment for findings
- Optional AI discovery from small security-relevant smali slices
- TXT, CSV, and PDF report export
- Settings for remote AI endpoint, API key, model, and chunk size

## Architecture

This repository contains a single Android module: `:app`.

Main components:

- UI entry point: `dev.shinyparadise.sast.ui.MainActivity`
- Application class: `dev.shinyparadise.sast.SASTApp`
- Dependency injection: Koin module in `AppModule`
- Pipeline orchestration: `AnalyzerInteractor`
- Rule scanner: `SmaliAnalyzer`
- AI discovery: `SmaliCandidateExtractor`, `SmaliVulnerabilityDiscoverer`, `AICoreSmaliDiscoverer`, `HeuristicSmaliDiscoverer`
- AI enrichment: `VulnerabilityAIAnalyzer`, `AICoreAnalyzer`, `RemoteAIAnalyzer`, `OnDeviceAIAnalyzer`
- Reports: `ReportGenerator`

## Build, Test, And Run

Compile Kotlin:

```bash
./gradlew :app:compileDebugKotlin
```

Run local unit tests and build the debug APK:

```bash
./gradlew :app:testDebugUnitTest :app:assembleDebug
```

Install the debug build:

```bash
./gradlew :app:installDebug
```

Start the app manually:

```bash
adb shell am start -n dev.shinyparadise.sast/dev.shinyparadise.sast.ui.MainActivity
```

## AI Analysis Modes

The app supports best-effort AI analysis paths:

- On-device AI through ML Kit GenAI Prompt API and AICore/Gemini Nano on supported devices
- Remote AI through an OpenAI-compatible endpoint configured in app settings
- Heuristic fallback when AI support is unavailable or unsuitable

Remote AI settings include URL, API key, model, and chunk size.

## Analysis Pipeline

1. The app receives an APK URI and copies it to a local file for analysis.
2. The APK is decompiled to smali.
3. `SmaliAnalyzer` runs deterministic rules.
4. Optional AI discovery extracts small security-relevant method slices.
5. AICore or heuristic discoverers inspect those slices.
6. Optional AI enrichment adds context to findings.
7. Findings are deduplicated and prepared for display.
8. `ReportGenerator` creates exportable reports.

Current deterministic rules cover:

- Hardcoded secret patterns
- Insecure crypto strings
- WebView JavaScript enabling in smali

## Reports

Reports can be exported as:

- TXT
- CSV
- PDF

Report output is generated by `ReportGenerator`. AI metadata in exports is still limited.

## Limitations

- This is not a mature production scanner.
- Rule coverage is currently narrow.
- AI discovery is best effort and can miss issues or produce noisy findings.
- On-device AI depends on device support.
- Remote AI requires user-configured endpoint credentials.
- Test coverage is currently minimal.
- Large APK handling can be improved, especially around smali file reading.

## Security And Privacy Notes

APK contents may contain sensitive code, strings, and metadata. Remote AI analysis can send selected smali slices or finding context to the configured endpoint. Use remote AI only with endpoints you trust.

API keys are configured in app settings. Treat them as secrets. On-device AI keeps analysis local when supported, but availability depends on the device and system services.
